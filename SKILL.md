# AI 日报 Skill

## 描述
一个面向产品经理和 AI 从业者的 AI 简报自动生成工具。用户只需一句指令，即可触发 AI 从预设信息源搜索近期 AI 领域资讯（默认过去 3 天），按规范筛选、提炼、排序后，自动生成结构化的 HTML 日报页面。

## 触发条件
当用户的请求涉及以下内容时，应激活此 Skill：
- 说出「生成今日简报」「生成 AI 日报」「今日 AI 简报」「AI 日报」等指令
- 提到「AI 新闻」「AI 资讯」「今天有什么 AI 动态」等关键词
- 要求搜索 AI 领域最新信息并整理

## 核心能力

### 执行流程

当用户触发 Skill 后，按以下步骤执行：

#### 第一步：解析用户指令
1. 识别是否有**临时新增信息源**（如用户说「顺便看看 xxx 网站」）
2. 识别是否有**特定关注方向**（如用户说「重点关注多模态」）
3. **识别时间区间**：
   - 如用户未指定时间范围 → 默认获取**过去 3 天**内发布的信息（含当天）
   - 如用户指定时间范围（如「过去 1 周」「最近 7 天」「2月1日到2月15日」）→ 按用户指定范围获取
   - 时间区间记为 `DATE_START` ~ `DATE_END`，用于后续搜索和页面展示
   - 示例：用户 2026-02-24 说「生成 AI 日报」→ 区间为 2026-02-22 ~ 2026-02-24
   - 示例：用户说「汇总过去 1 周的信息」→ 区间为 2026-02-18 ~ 2026-02-24

#### 第二步：信息搜索与采集
1. 加载固定信息源列表（`references/info-sources.md`）
2. 如有用户临时新增的信息源，合并到搜索列表
3. **按优先级顺序搜索**：
   - P0（必查）：官方一手信息源 + 顶级研究者
   - P1（选查）：深度访谈播客 + 前沿资讯
4. 使用 `web_search` 工具对每个信息源进行搜索，关键词格式：`{信息源名称} AI latest news {DATE_START}..{DATE_END}`
5. 使用 `web_fetch` 工具获取搜索到的重要页面内容
6. 采集每条信息的原始发布时间，格式统一为 YYYY-MM-DD；如无法确定精确日期，标注为「近期」
7. **时间过滤**：丢弃发布时间明确在时间区间（`DATE_START` ~ `DATE_END`）之外的信息

#### 第三步：信息筛选与过滤
严格按以下规则执行：

**时效性规则（最重要）：**
- **只收录确认在时间区间内实际发布/公布的信息**
- 每条信息的发布日期必须可从原始来源确认（官方博客时间戳、社交媒体发布时间、新闻报道明确日期）
- 严禁收录：
  - 无法确认具体发布日期的信息
  - 发布日期在时间区间之前的旧闻（即使最近才被广泛讨论）
  - 无法确认是否真实存在的信息
- **宁缺毋滥**：符合条件的高质量信息不足 3 条，就只返回实际找到的条数，不要凑数

**自动过滤规则：**
- 重复信息：保留最源头的信息来源（优先级：官方 > 研究者 > 访谈 > 资讯），删除二次搬运
- 质量过滤：仅保留满足以下条件中**任意 2 条及以上**的内容：
  - 来自官方/顶级研究者（优先级最高）
  - 有明确技术/产品突破（核心筛选条件）
  - 能直接落地/复现（实操性优先）
  - 影响行业方向（趋势性优先）

**信息处理要求：**
- 核心内容控制在 50-100 字，精准提炼核心要点
- 按「信息价值」排序：官方一手 > 研究者技术 > 访谈洞察 > 资讯热点
- 每条信息必打标签，仅使用指定标签：`#技术` `#产品` `#应用` `#论文` `#行业`（可组合，1-3个）

#### 第四步：生成 HTML 日报
1. 读取 HTML 模板（`assets/template.html`）
2. 将筛选后的信息填入模板，包括每条信息的发布时间
3. 填入信息覆盖区间（`{{DATE_START}}` ~ `{{DATE_END}}`）
4. 生成的 HTML 文件保存到：`/Users/jieqiongwu/CodeBuddy/AI简报/YYYY-MM-DD-AI日报.html`
5. 文件名中的日期为**生成日期**（即 DATE_END），格式严格为 `YYYY-MM-DD-AI日报.html`（如 `2026-02-23-AI日报.html`）

#### 第五步：输出与确认
1. 将生成的 HTML 文件复制到当前工作空间目录，启动本地 HTTP 服务（如 `python3 -m http.server`），并使用 `preview_url` 工具打开预览
2. **回复用户时必须包含可点击的预览链接**（如 `http://localhost:端口/文件名.html`），方便用户直接访问
3. 告知用户日报已生成，显示文件路径
4. 列出今日收录的信息概要（标题 + 标签）
5. 标注「今日深度学习精选」是哪一条
6. **如发现优质的非固定信息源**（某个网站/频道/博客在本次搜索中贡献了高价值信息），主动提醒用户：「发现优质信息源 XXX，建议添加到固定信息源库中，是否需要我帮你添加？」
7. 询问用户是否需要调整或补充

## 输出格式规范

每条信息必须包含以下字段：

```
### 【信息N】
- **标题：**（简洁明了，含核心关键词）
- **发布时间：**（YYYY-MM-DD，如 2026-02-21；无法确定精确日期时标注「近期」）
- **核心内容：**（50-100字，客观提炼，不添加个人观点）
- **价值：**（明确说明信息价值，如「可落地到XX场景，提升XX效率」）
- **标签：**（#技术 #产品 等，1-3个）
- **链接：**（最源头链接，优先官方发布页，页面直接展示完整 URL）
```

## 标签规范
- 严格使用 5 个指定标签：`#技术` `#产品` `#应用` `#论文` `#行业`
- 技术类信息必带 `#技术`，产品类必带 `#产品`
- 可组合使用（如 `#技术 #产品`），每条 1-3 个标签

## 链接规范
- 优先官方链接，避免跳转链接
- YouTube 链接直接粘贴原视频地址
- **页面展示时直接显示完整 URL**（不使用「查看原文」等隐藏链接文字），方便用户一眼识别信息来源出处
- **非固定信息源标记**：如果信息的来源链接域名不在 `references/info-sources.md` 固定列表中，需在链接后添加 `📡 非固定源` 黄色标记（class: `source-external`），方便用户快速识别哪些信息来自库外来源

## 内容规范
- 核心内容不添加个人观点，仅客观提炼
- 价值部分需明确具体，不模糊（如不说「有价值」，要说「可落地到 AI 创作场景，提升工作效率」）
- 每日建议收录 3-10 条，不凑数、保质量

## 分类展示规范
信息按以下三大分类分组展示，每组有独立标题和图标：

1. **🔬 技术突破**：标签含 #技术 或 #论文 的信息（模型架构、芯片硬件、训练方法、安全研究等）
2. **🚀 产品应用**：标签含 #产品 或 #应用 的信息（新产品发布、功能更新、应用落地等）
3. **📊 行业动态**：标签含 #行业 的信息（融资投资、政策合规、战略合作、市场格局等）

**归类规则：**
- 每条信息仅归入一个最匹配的分类，不重复出现
- 若同时含多个标签，按第一标签归类（如 #技术 #产品 归入「技术突破」）
- 各分类内部按信息价值排序
- 分类标题显示该类信息条数

## 网页风格
- 浅色简洁风：白色背景，类似 Notion / Newsletter 风格
- **分类分组布局**：信息按「技术突破」「产品应用」「行业动态」三大板块分组展示，每组有彩色分隔线标题
- 卡片式布局展示每条信息
- 标签使用彩色小标签样式
- 响应式设计，适配手机端浏览
- 不需要英文摘要

## 定时生成 & 推送

支持定时自动生成 AI 简报并推送到微信/企业微信。

### 快速设置

```bash
cd automation
bash setup.sh        # 安装依赖 + 生成配置
vim config.yaml      # 填写 API Key 和推送配置
python3 auto_brief.py --no-push   # 测试生成
python3 auto_brief.py             # 生成并推送
```

### 推送渠道

| 渠道 | 适用场景 | 设置难度 |
|------|---------|---------|
| 企微群机器人 Webhook | 推送到企微群聊 | ⭐ 最简单 |
| 企微自建应用消息 | 推送到个人企微 | ⭐⭐ 中等 |
| PushPlus | 推送到个人微信 | ⭐ 简单 |
| Server酱 | 推送到个人微信 | ⭐ 简单 |

### 定时任务

- **crontab**：`0 9 * * * cd /path/to/ai-daily-brief && python3 automation/auto_brief.py`
- **macOS launchd**：`launchctl load ~/Library/LaunchAgents/com.ai-daily-brief.plist`

详见 `automation/config.yaml.example` 中的配置说明。
